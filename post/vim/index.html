vim编辑利器

vim编辑器的简洁清晰明了的风格，强大的编辑能力，以及高效的coding，就是一款为程序员而生的编辑器，作为一个程序员，也应该学会使用vim。但是想用好vim也不是一件简单的事情，当你用好vim后带来的好处也是很多的，当然，”工欲善其事，必先利其器”，学习vim肯定是不错的。

vim的安装与配置

我的macos系统是自带的vim编辑器，当然也可以安装最新版的vim

首先去下载源码https://github.com/vim/vim

然后正常安装

$cd src
$./configure
$make && make install

就安装好了，可以运行vim检查是否安装成功。

vim的配置主要是修改vim的配置文件～／.vimrc，这个可以百度到很详细的配置，在这里就不赘述了。

vim插件

vim插件管理器

vundle是一款特别好用的vim插件的管理器，安装也是非常简单。

git clone https://github.com/gmarik/vundle.git ~/.vim/bundle/vundle

然后修改配置文件~/.vimrc

filetype off
set rtp+=~/.vim/bundle/vundle/
call vundle#rc()
Bundle 'gmarik/vundle'
Bundle '其他的 github 位置...'
filetype plugin indent on

vundle本身也是一个插件，所以也需要安装，目前vundle这个项目托管在github上，地址是

https://github.com/gmarik/vundle

所以上面配置文件中加上了这个插件的安装

最后打开vim，输入:BundleInstall会显示安装的过程，done代表安装完成。

source instght类

ctags

Ctags工具是用来遍历源代码文件生成tags文件，这些tags文件能被编辑器或其它工具用来快速查找定位源代码中的符号（tag/symbol），如变量名，函数名等。比如，tags文件就是Taglist和OmniCppComplete工作的基础。

安装

我的macos系统下直接使用homebrew

brew install ctags

也可以直接用源码安装，最好下载EXUBERANT ctags，这个提供了比较强大的基础功能。正常安装

cd src
./configure
make && make install

常用命令

  1. ctags -R * 对当前目录下所有的文件包含子目录下的文件进行递归创建tag文件，当vi运行后会自动加载tag文件中所有的对象包含
  • 用#define定义的宏
  • 枚举型变量的值
  • 函数的定义、原型和声明
  • 名字空间（namespace）
  • 类型定义（typedefs）
  • 变量（包括定义和声明）
  • 类（class）、结构（struct）、枚举类型（enum）和联合（union）
  • 类、结构和联合中成员变量或函数
  1. vi -t tag tag为变量，函数或者其他，就是对当前目录下进行tag对象查找，找到就打开文件并定位到这一行。
  2. :ts 如果函数或者变量有在这个项目中有多处定义的话，用这个命令列出来可供选择
  3. :tp 上面列表当前选择的上一个
  4. :tn 列表当前选择的下一个
  5. Ctrl+] 查找当前变量或者函数的定义处并打开定位
  6. Ctrl+t 回退Ctrl+]的操作，都可以多处查找

taglist

TagList插件,是一款基于ctags,在vim代码窗口旁以分割窗口形式显示当前的代码结构概览,增加代码浏览的便利程度。

安装

  1. 下载安装包http://www.vim.org/scripts/script.php?script_id=273
  2. 进入~/.vim目录，将Taglist安装包解压，解压后会在~/.vim目录中生成几个新子目录，如plugin和doc（安装其它插件时，可能还会新建autoload等其它目录）。
  3. 进入~/.vim/doc目录，在Vim下运行”helptags .“命令。此步骤是将doc下的帮助文档加入到Vim的帮助主题中，这样我们就可以通过在Vim中运行“help taglist.txt”查看taglist帮助。
  4. 修改配置文件,可以根据需要修改，不修改也可以

基本用法

  1. 打开与关闭 :Tlist 可以在vim配置中设置快捷键map :TlistToggle
  2. 左右窗口切换 Ctrl+w左右，也可以用Ctrl+h,l
  3. u更新tag 其他的我觉的用处不大，需要的适合了解一些就好。

cscope

Cscope是一个类似ctags的工具。 你可以把它想作是超过频的ctags，因为它功能比ctags强大很多。

安装

macos上直接使用

brew install cscope

就好。至于源码安装和上面的ctags一样，这边就不多说了。

基本使用

首先在目录下使用“cscope -Rqb”生成cscope.out文件;

使用时先在VIM的命令行下调用“cscope add/../../cscope.out”（或cs add /../../cscope.out)命令添加一个cscope数据库，然后就可以调“cscope find”（或 cs f ）命令进行查找了。

VIM支持8种cscope的查询功能，如下：

  • c:Find functions calling this function//查找调用本函数的函数
  • d:Find functions called by this function //查找本函数调用的函数
  • e:Find this egrep pattern//查找egrep模式，相当于egrep功能，但查找速度快多了
  • f:Find this file //查找并打开文件，类似vim的find功能
  • g:Find this definition//查找函数、宏、枚举等定义的位置，类似ctags的功能
  • i:Find files #including this file //查找包含本文件的文件
  • s:Find this C symbol //查找C语言符号，即查找函数名、宏、枚举值等出现的地方
  • t:Find assignments to //查找指定的字符串

MarkDown类

vim-markdown

vim-markdown是md文档高亮方案的插件，通过vundle安装

Plugin 'godlygeek/tabular'
Plugin 'plasticboy/vim-markdown'

然后运行BundleInstall安装就可以，运行vim打开md文档就可以看到高亮了。

vim-instant-markdown

这是一个实时预览的插件，当你用vim打开markdown文档的时候，会自动打开一个浏览器窗口，并且可以实时预览。此插件目前只支持OSX 和 Unix/Linuxes操作系统。

安装之前需要安装nodejs和npm

brew install node

就都安装好了，npm是一款nodejs包的管理工具。然后还是通过vundle来安装

Plugin 'suan/vim-instant-markdown'

安装好后编辑md文件就会实时的打开一个浏览器窗口显示文档内容，其实就是访问本地的http://localhost:8090

vim使用

恢复文件

当文件还没有保存就意外关闭时，只要用-r就行

vi -r filename

然后删除.swp文件。

打开多个文件

  1. 直接用vim file1 file2…filen
  2. 打开vim，然后用open file来打开

同时显示多个文件：

其实就是分屏

split，横分屏

vsplit竖分屏

窗口切换用Ctrl+w左右键，或者Ctrl+h，l

文件切换

bn／ctrl+6 下一个文件

bp 上一个文件

光标操作

翻页 Ctrl+f 向下 Ctrl+b 向上 翻半页 Ctrl+d Ctrl+u

编辑

撤销 u 恢复 Ctrl+r 打开文件 open name

set nu 或者 set number //显示行号；

set nonu 或者 set nonumber //关闭自动显示行号；

set ic //查找时忽略大小写；

set noic //查找时不忽略大小写。
