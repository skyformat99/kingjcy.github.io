systemctl和systemd

Systemctl是一个systemd工具，主要负责控制systemd系统和服务管理器。

Systemd是一个系统管理守护进程、工具和库的集合，用于取代System V初始进程。Systemd的功能是用于集中管理和配置类UNIX系统。

基础使用

  • systemctl –version 查看版本
  • systemctl start/stop/restart/reload/status firewalld.service 启动，停止，重启重载，检查某个单元是否运行
  • systemctl list-unit-files –type=service 列出所有的服务
  • systemctl enable/disable http.service 开机启动禁止服务
  • systemctl kill http 杀死某个服务
  • systemctl show http 检查某个服务的配置

将一个新服务加到开机启动中，在centos 6的system v中是使用chkconfig etcd on,而在centos 7的systemd中使用的是systemctl enable etcd.service

都需要一个配置文件，centos 7下在/lib/systemd/system/下，这个是系统的，如果是用户的，则在对应的用户里

[Unit]              服务的说明   
Description=etcd    描述
After=              服务类别

[Service]           服务的参数
Type=forking        后台运行的方式 
Environment=ETCD_NAME=kubernetes  可以这样设置环境变量
ExecStart=/usr/lib/etcd         启动命令，必须是绝对路径 
ExecReload=
ExecStop=
PrivateTmp=true  

[Install]       安装的设置
WantedBy=multi-user.target  多用户模式
